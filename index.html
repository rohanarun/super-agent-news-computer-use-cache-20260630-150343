How a computer-use cache works in practice

1. Run the task once

Super performs real computer actions in a cloud browser or app — clicking, typing, navigating, and waiting for results.

2. Actions are saved

Successful action traces are stored as a reusable computer-use cache instead of being thrown away.

3. Replay on repeat

Future runs replay cached actions directly, avoiding repeated planning and model calls.

4. Heal when needed

If a page layout changes, Super can fall back to a fresh run and update the cache.
